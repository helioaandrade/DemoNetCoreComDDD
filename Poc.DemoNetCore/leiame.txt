
-------------------------------------
Projeto Domonstração NetCore com DDD
-------------------------------------

Este projeto foi criado com Net Core com as seguintes caracteristicas:
- Abordagem DDD
- Web API com autenticação usando JWT
- Banco de dados SQlite(mas pode ser usado qualquer banco, pois a camada de repositório é agnóstica)
- ORM Entitity Framework Core(possibilidade de usar o SqlServer/MongoDb/Micro ORM Dapper, etc)
- Uso de Padrões IOC + DI  
- Swagger integrado no projeto da camada da API para facilitar os testes das APIs
- Variáveis de configuração estão definidas no arquivo AppSettings.Json
- Testes unitários na camada Web Api(Utilização do framework NUnit)

- Programa Console para testar as chamadas das Apis( foi utizado o componente RestSharp para consumir os serviços das APis)
   - install-package restsharp no projeto ConsoleTeste

------------------------------------
Testes Unitários
-------------------------------------
Os Testes Unitários tem o objetivo de testar as APis, e para tanto é necessário que a API esteja "Rodando"
Deve-se dar um "Build" no Projeto API e em seguida "start" o serviço, usando o comando "dotnet Api.dll", como exibido abaixo:

Por exemplo:
cd C:\Poc\Poc.DemoNetCore\src\Api\bin\Debug\netcoreapp2.0
dotnet Api.dll

-----------
Observaçoes
-----------
(1) Foi usado o aplicativo SQLiteStudio que pode se baixado em https://sqlitestudio.pl/index.rvt?act=download
(2) O banco de dados está na pasta DemoNetCoreComDDD/Poc.DemoNetCore/db/demonetcore.db
(3) Os scripts de banco de dados estão na pasta DemoNetCoreComDDD/Poc.DemoNetCore/Scripts/
 

--------------------------------------
Procedimentos para executar os testes
--------------------------------------
1) Configurar o projeto API(Pasta Serviços/Api) com "SET AS Startup Project)
   Automaticamente, abrirá um Browser no endereço localhost:8090/swagger
   O projeto da API com o Swagger integrado visa facilitar os testes das APIS(Opcional)
